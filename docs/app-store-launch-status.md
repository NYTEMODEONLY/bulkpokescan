   - `~/.claude/plans/great-now-i-frolicking-hare.md` (full plan)
2. **Confirm icon decision** — see the open question above. Either
   accept v2, request a tweak, or ask for a different concept entirely.
3. **Commit the v2 icon** once approved:
   ```
   git add ios/BulkPokeScan/Info.plist ios/BulkPokeScan/Resources/Assets.xcassets/AppIcon.appiconset/icon-1024.png
   git commit -m "App icon v2: card + QR (no Pokéball reference)"
   git push
   ```
4. **Verify a clean Release build** before TestFlight:
   ```
   cd ios
   xcodebuild -project BulkPokeScan.xcodeproj -scheme BulkPokeScan \
     -configuration Release -destination 'generic/platform=iOS' \
     archive -archivePath ./build/BulkPokeScan.xcarchive
   ```
5. **TestFlight** — upload via Xcode (Window → Organizer → Distribute App)
   or Transporter. Add yourself as Internal Tester. Verify on device.
6. **Screenshots** — boot iPhone 16 Pro Max simulator (1290×2796),
   capture 6 frames per the spec in `docs/app-store-connect-metadata.md`,
   add marketing overlays in Figma.
7. **App Store Connect** — create app record, paste metadata, upload
   build + screenshots, submit.

## Trademark posture (don't lose track of this — single biggest review risk)

Apple's guideline 5.2.1 (intellectual property) is the most likely
rejection vector for a Pokémon-adjacent app. Mitigations already
shipped:

- App name "BulkPokeScan" uses "Poke" (no trademark) instead of "Pokémon"
- v2 app icon has zero Pokéball reference (card + QR composition)
- Trademark line "Pokémon is a trademark of Nintendo, Creatures Inc.,
  and GAME FREAK Inc. — this app is unofficial and unaffiliated"
  appears in: App Store description, in-app About, Settings → Legal
  footer, /privacy page, /terms page
- App Review notes explicitly call out that the app does not connect
  to or interact with any Pokémon-branded service, does not generate
  / validate / redeem / share / trade codes, and only ships a
  productivity utility for organizing user-owned codes

If 5.2.1 rejection arrives anyway: it's a winnable resolution-center
round trip with the trademark + nominative-fair-use argument.
